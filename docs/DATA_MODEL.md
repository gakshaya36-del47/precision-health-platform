# Data Model — Version 1

## Purpose

The data model provides a structured representation of user-provided information for the precision health research prototype.

The model separates input storage from downstream calculations and evidence interpretation.

No disease diagnosis or composite risk score is generated by the data model itself.

## Profile Structure

### UserProfile

The top-level profile contains:

- age
- sex
- country or region
- anthropometric measurements
- lifestyle information
- family history
- limited clinical context

### Anthropometrics

The current structure stores:

- height
- height unit
- weight
- weight unit
- waist circumference
- waist measurement unit

These values are inputs for future classification/assessment functions.

### LifestyleProfile

The current structure stores:

- physical activity frequency
- dietary pattern
- smoking status
- alcohol status
- sleep duration
- perceived stress

These variables are stored as contextual inputs for future validated instruments and evidence mapping.

They do not independently produce a disease-risk score.

### FamilyHistory

The current structure stores self-reported family history related to:

- diabetes
- cardiovascular disease

### ClinicalContext

The current structure stores:

- use of blood pressure medication
- history of elevated blood glucose

These fields are intended for future validated instrument implementation where appropriate.

## Validation

Basic input validation is currently implemented.

The validation layer checks:

- age plausibility
- allowed sex categories
- positive anthropometric values
- plausible sleep-duration values

Validation errors are returned as a list rather than raising clinical conclusions.

## Scientific Boundary

The data model is not a clinical model.

It does not:

- diagnose disease
- calculate a composite health score
- estimate cardiovascular risk percentages
- infer genetic susceptibility
- treat lifestyle factors as deterministic causes
- replace validated clinical assessment

## Architectural Principle

The data model stores inputs.

Validated screening instruments, classification functions, and evidence-mapping modules must remain separate from the profile data structures.

This separation is required by the Version 1 Evidence Freeze documented in:

`docs/MODEL_AND_EVIDENCE_VERIFICATION.md`

## Current Status

**Implemented:**

- structured profile dataclasses
- basic input validation
- automated tests

**Not yet implemented:**

- BMI calculation
- waist-to-height ratio calculation
- FINDRISC scoring
- PHQ-9 scoring and safety pathway
- cardiovascular evidence mapping
- cross-domain evidence mapping
